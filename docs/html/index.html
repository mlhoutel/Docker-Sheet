Desktop Languages

I describle desktop languages as languages which are mainly used to create application software on computer / mobile platforms independently of web (warning, not exclusive: for example, python can be used for the web, like currently this website 😉 )

That include languages that:

  • need to be compiled in machine language for the plateform (like C, C++, C# …)

  • need to be compiled on the fly by a virtual machine (like Java, Scala …)

  • are translated in read time, or scripting languages (like Python, Ruby …)
